## 1. 프로덕트 개요 — "얼마였지?"

### 앱 이름 및 핵심 메시지

> **"마트의 가짜 세일에 속지 마세요"**

**앱 이름**: 얼마였지?

마트에서 "특가! 30% 할인!" 팻말을 보는 순간 드는 의심 — *"진짜 싼 건가? 평소엔 얼마였지?"*
이 질문에 바코드 스캔 한 번으로 즉답을 주는 앱.

---

### 타깃 사용자

| 세그먼트 | 핵심 니즈 |
|---|---|
| **주부** (30~50대) | 장볼 때 마트 할인 진위 판별, 온라인 vs 오프라인 어디가 더 저렴한지 |
| **1인 가구** (20~30대) | 편의점·마트·쿠팡 중 실시간 최저가 즉시 비교 |
| **스마트 컨슈머** (전 연령) | 내가 지난번 봤던 가격보다 지금 더 싼지 판단 |

→ **전 국민 타깃**: 마트에서 쇼핑하는 모든 사람

---

### 해결하는 문제

**문제 1: 마트의 가짜 세일**
- 마트가 기준가를 높여놓고 "20% 할인" 표시 → 실제로는 평소 온라인 가격과 동일하거나 더 비쌈
- 소비자는 할인 팻말만 보고 구매 → 사실 손해

**문제 2: 온/오프라인 최적 구매 결정 불가**
- 마트 선반 앞에 서서 "이걸 쿠팡에서 시키는 게 나은가?" 판단 불가
- 배송비 포함 실질 가격 비교 도구 부재

**문제 3: 내 구매 기억의 불확실성**
- "이 샴푸, 지난번엔 얼마였지?" → 기억에 의존할 수밖에 없음
- 앱이 내 과거 스캔 가격을 기억하고 변동을 알려줌

---

### 핵심 경험 흐름

```
마트 진열대 앞에 선다
        ↓
앱 실행 → 즉시 카메라(스캐너)가 열림 (Type 1 UX)
        ↓
바코드에 갖다 대면 → 2초 이내 결과
        ↓
┌──────────────────────────────────────┐
│  세탁세제 OO 2L                      │
│                                      │
│  🔴 마트 현재가: 12,900원            │
│  ✅ 쿠팡 최저가: 9,800원 (-24%)     │
│  📅 내 이전 스캔가: 10,500원 (89일 전) │
│                                      │
│  → 쿠팡에서 사는 게 3,100원 이득    │
│  → 마트 할인가인데도 온라인보다 비쌈  │
└──────────────────────────────────────┘
        ↓
구매 결정 또는 내 기록으로 저장
```

---

### 비즈니스 모델

**하이브리드 수익 모델** (Freemium 구독 없음)

| 수익원 | 방식 | 예상 비중 |
|---|---|---|
| **인앱 광고 (AdMob)** | 하단 배너 광고 + 전면 광고 (스캔 후 결과 화면 사이) | 40% |
| **제휴 마케팅 수수료** | 쿠팡 파트너스, 네이버 쇼핑 파트너 링크 아웃 | 60% |

> 구독 모델을 배제한 이유: 가격비교 앱의 핵심 사용자(주부, 스마트 컨슈머)는 월정액 결제 허들에 민감. 광고 수익 + 제휴 커미션 하이브리드가 초기 MAU 성장에 유리.

## 2. 글로벌 시장 현황

### 시장 규모 및 성장률

| 시장 | 2024년 | 2025년 | 2032/2033년 | CAGR |
|---|---|---|---|---|
| 가격비교 앱 | $383M | $410M | $676M | 7.36% |
| 바코드 스캐너 전체 | $92.2억 | $98억 | $168억 (2036) | 6.7% |
| 모바일 커머스 | $1.7조 | **$2.5조** | — | — |
| 아시아태평양 가격비교 | $116억 (2023, 39.2% 점유) | — | — | — |

### 시장 성장 동인

1. **인플레이션 지속** → 소비자 가격 민감도 전례없이 상승, "가성비" 문화 전 계층 확산
2. **모바일 커머스 폭발** → 전체 이커머스 트래픽의 50%+ 이미 모바일 경유
3. **AI/ML 기술 성숙** → 바코드 인식 정확도 98%+ 달성, 처리 속도 30% 향상
4. **APAC 급성장** → 글로벌 시장의 39.2% 차지, 한국/일본/중국 중심 고성장

### 시장 기회 요약

- 가격비교 앱을 쓰는 모바일 쇼핑 유저 중 **대부분이 속도와 정확도에 불만족**
- 기존 앱은 "시장 데이터"를 보여주는 도구 — "나만의 기록"을 보여주는 앱은 전무
- 한국은 온라인 소매 비중 세계 1위(28.2%)인데 모바일 가격비교 앱 품질은 낙후

## 3. 경쟁 앱 포지셔닝 분석

### 3-1. 경쟁 앱 유형별 포지셔닝 맵

시장에 존재하는 앱들을 **"스캔 방식"** × **"비교 대상"** 두 축으로 분류하면, 우리 앱이 차지할 수 있는 공백이 명확해진다.

```
                   [온라인 가격만 비교]
                          │
           폴센트 ────────┤──── 역대가
           (쿠팡 전용)   │    (쿠팡 히스토리)
                          │
[검색/수동 입력] ─────────┼───────────────── [바코드 스캔]
                          │
           에누리 ────────┤──── ShopSavvy
           다나와         │    (글로벌)
                          │
                   ★ 얼마였지? ★
            (스캔 + 온라인 비교 + 내 이전 오프라인 가격)
                          │
                   [온+오프라인 통합 비교]
```

---

### 3-2. Price Note — 가장 유사한 개념의 경쟁앱, 그러나 다른 방향

**Price Note란?**
- 사용자가 마트에서 본 가격을 **직접 손으로 입력**하여 기록·분석하는 앱
- 가격 히스토리 추적, 상품별 기록 관리에 특화된 "디지털 가계부" 스타일 앱

**Price Note의 강점**
- 오프라인 가격을 직접 기록하고 싶은 니즈를 정확히 겨냥
- 충성도 높은 유저 기반 (직접 입력하는 사용자 = 진지한 절약 실천자)
- 개인 가격 히스토리 개념을 먼저 실증

**Price Note의 허들 — 우리가 해결하는 것**

| Price Note의 문제 | 얼마였지?의 해결 |
|---|---|
| 가격을 **손으로 직접 입력**해야 함 → 귀찮아서 포기 | 바코드 스캔 한 번 → 자동 기록 |
| 온라인 최저가와 **즉시 비교 불가** | 스캔 즉시 쿠팡/네이버 실시간 가격 표시 |
| "지금 사야 하나?" 판단에 도움 안 됨 | 온라인 최저가 vs 현재 오프라인 가격 즉시 대조 |
| 앱을 켜서 상품 찾고 입력하는 **3~5단계** | 앱 실행 → 카메라 즉시 → 스캔 **1단계** |

> **포지셔닝 요약**: Price Note가 "기록의 도구"라면, 얼마였지?는 "즉각적인 구매 결정 도우미"다. 가격 기록은 스캔 시 자동으로 쌓이는 부산물이다.

---

### 3-3. 경쟁 앱 유형별 비교표

| 앱 | 바코드 스캔 | 온라인 실시간 비교 | 내 과거 가격 기록 | 오프라인 스캔 즉시 판단 | 주요 한계 |
|---|---|---|---|---|---|
| **에누리** | ❌ | ✅ | ❌ | ❌ | PC 중심, 모바일 UX 처참 |
| **다나와** | ❌ | ✅ (전자제품) | ❌ | ❌ | 전자제품 외 취약, 느림 |
| **폴센트** | ❌ | ✅ (쿠팡만) | ❌ | ❌ | 쿠팡 단독, 바코드 없음 |
| **역대가** | ❌ | ✅ (쿠팡만) | ❌ | ❌ | 쿠팡 단독, 기능 단순 |
| **Price Note** | ❌ | ❌ | ✅ (수동 입력) | ❌ | 입력 허들, 온라인 비교 없음 |
| **ShopSavvy** | ✅ | ✅ (글로벌) | ❌ | ⚠️ 부정확 | 30초+ 느림, 한국 미지원 |
| **일반 QR/바코드 스캐너** | ✅ | ❌ | ❌ | ❌ | 가격 정보 없음 |
| **네이버 앱 (렌즈)** | ✅ | ✅ (제한적) | ❌ | ❌ | 가격비교 기능 제거 |
| **★ 얼마였지?** | ✅ | ✅ (멀티) | ✅ (자동 기록) | ✅ | — |

> **공백 발견**: 바코드 스캔 + 온라인 실시간 비교 + 내 과거 가격 자동 기록 세 가지를 동시에 갖춘 앱이 **한국 시장에 전무**하다.

## 4. 글로벌 경쟁사 심층 분석

### 4-1. ShopSavvy (미국 1위 바코드 가격비교 앱)

**기본 정보**
- 커버 리테일러: 60,000개 이상 (온/오프라인 포함)
- 앱스토어 평점: iOS 4.5+ / Android 4.5+
- 특징: ML 기반 스캔, 가격 히스토리 차트, AI 어시스턴트, TLDR 리뷰 요약

**강점**
- 가장 많은 리테일러 커버리지 (60,000개+)
- 가격 워치리스트 기능 + 가격 하락 시 이메일/푸시 알림
- 각 리테일러별 가격 히스토리 차트 제공
- AI 기반 제품 질의응답 기능 (최신 업데이트)

**치명적 약점 — 실제 사용자 리뷰 기반**

| 문제 | 구체적 사례 |
|---|---|
| **극도로 느린 속도** | 스캔 후 결과까지 30초~1분+ 대기. "구글에 UPC 직접 검색하는 게 더 빠름" |
| **앱 크래시** | 스캔한 제품의 약 95%에서 "Searching for prices" 화면 후 홈 화면으로 강제 종료 |
| **가격 정확도 심각** | Amazon 표시가 $9.45인데 실제 가격은 $4.48 (2배 오차). "실제 매장 가면 가격이 다름" |
| **오프라인 결과 신뢰 불가** | Walmart, Target 등 대형 오프라인 마트 가격은 거의 틀림. 온라인 중심으로만 작동 |
| **광고 과다** | 광고가 기능을 방해하는 수준. 광고 로딩으로 체감 속도 더 저하 |
| **스캔 불안정** | 바코드를 완전히 정지 상태로 수 초간 유지해야 인식. 조금이라도 흔들리면 실패 |
| **품질 하락** | 최근 회사 인수 후 기존 기능 퇴행. 구독 취소가 의도적으로 어렵게 설계됨 |

**평점과 실제의 괴리**: 4.5점이지만 실제 리뷰는 "앱이 작동을 안 함", "사기 수준" 등 1~2점 리뷰 다수 → 리뷰 관리 의혹

**치명적 약점 — 실제 사용자 리뷰 기반 (Trustpilot)**

| **UI 업데이트 역주행** | 업데이트 후 검색 결과가 광고로 가득 참 |

### 4-4. CamelCamelCamel / Keepa (Amazon 가격 추적 전문)

**성공 요인 — 우리가 배워야 할 점**
- Amazon이라는 단일 플랫폼에 집중 → 깊은 데이터, 높은 정확도
- 가격 차트가 "언제 사야 하는지"를 직관적으로 보여줌
- Keepa: 계정 없이 즉시 사용 가능 → 가입 장벽 없음 → 우리 앱도 이 원칙 적용

**한계**
- Amazon 전용 — 한국 쿠팡/네이버 전혀 지원 안 함
- 바코드 스캔 기능 없음 — 오프라인 쇼핑과 연결 불가

## 7. 경쟁사 공통 약점 & 시장 공백

### 7-1. 공통 약점 Top 5 — 우리의 기회

#### 약점 1: 속도 (치명적 문제)

| 경쟁사 | 결과 속도 |
|---|---|
| ShopSavvy | 30초 ~ 1분 이상 |
| 에누리 | 인앱 브라우저 고정 (사실상 웹 속도) |
| 다나와 | 스크롤조차 느림 |
| **우리 목표** | **캐시 기준 1초, 최대 2초 이내** |

**왜 경쟁사들이 느린가**: 여러 쇼핑몰 API를 **순차적**으로 호출.

**우리의 해결책**: **병렬 API 호출** + **Redis 캐싱**. 쿠팡/네이버/G마켓을 동시에 호출하고, 인기 상품은 캐시에서 즉시 반환.

---

#### 약점 2: 가격 정확도 (신뢰 문제)

| 경쟁사 | 정확도 |
|---|---|
| PriceSpy | 실사례: 77% 가격 차이 |
| Idealo | 공식 약 80% (20% 오류율 인정) |
| 에누리 | 다수 불일치 보고 |
| ShopSavvy | Amazon 표시 $9.45 → 실제 $4.48 (2배 오차) |
| **우리 목표** | **공식 API 기준 85~90% + "조회 시각" 명시** |

**왜 경쟁사들은 공식 파트너 API를 안 쓰나?**

핵심은 하나다. **더 많은 쇼핑몰을 커버하고 싶어서.**

1. **커버리지 문제 (주된 이유)**: 11번가, SSG닷컴, 롯데온처럼 공식 API를 제공하지 않는 쇼핑몰도 포함하려면 크롤링이 유일한 방법. ShopSavvy처럼 60,000개 리테일러 커버리지를 광고하려면 크롤링 없이는 불가능. 그 대가로 정확도를 희생.
2. **Rate Limit**: 쿠팡·네이버 등 공식 API는 요청 수 제한이 있음. 수백만 MAU 규모가 되면 병목이 생김. 크롤링은 서버 증설로 해결 가능.

> **참고**: "파트너 API 승인 조건"과 "비즈니스 모델 충돌"은 실제로 큰 장벽이 아님. 쿠팡 파트너스와 네이버 쇼핑 파트너는 개인도 신청 가능하고, 두 파트너십을 동시에 가질 수 있음. 각각의 링크에서 커미션을 받으면 됨. Amazon PA API는 예외적으로 실적 조건이 있어 Phase 3까지 미루는 것이 적절.

**우리의 전략**:
- MVP: 쿠팡 파트너스 API + 네이버 쇼핑 API 두 개에 집중. 커버리지 좁은 대신 정확도 확보.
- API 없는 쇼핑몰(11번가, SSG 등)은 Phase 2에서 크롤링으로 추가.
- 캐시 TTL 짧게 유지 (인기 상품 15~30분).
- "OO분 전 기준 가격입니다. 구매 전 확인하세요" 조회 시각 명시.
- 모든 가격에 실제 쇼핑몰 링크 연결 → 최신 가격 최종 확인 유도.

---

#### 약점 3: "나만의 가격 이력" 부재 (절대적 차별점)

> **전 세계 어떤 경쟁사도 이 기능이 없다**

- 모든 경쟁사의 가격 히스토리 = "시장 전체의 가격 변동 데이터"

## 8. 우리 앱의 핵심 차별화 전략 (USP)

### 8-1. 핵심 메시지

> **"마트의 가짜 세일에 속지 마세요"**
>
> *바코드 하나면 진짜 최저가를 2초 안에 알 수 있습니다.*

이 메시지가 강력한 이유:
- **대중적 공감**: "마트 할인에 속았다"는 경험은 거의 모든 소비자가 가지고 있음
- **명확한 적**: "가짜 세일" → 구체적인 문제 정의
- **즉각적 해결책**: "바코드 하나면" → 액션이 단순함
- **시간 명시**: "2초 안에" → 속도라는 경쟁우위를 메시지에 담음

---

### 8-2. USP 1: Type 1 UX — 앱 실행 즉시 카메라

**Type 0 UX (기존 경쟁사 방식)**
```
앱 아이콘 클릭
    → 홈 화면 로딩 (배너 광고 노출)
    → "스캔" 버튼 찾아서 클릭
    → 카메라 권한 요청 팝업 (첫 실행 시)
    → 카메라 열림
```
총 4~5단계, 5~8초 소요

**Type 1 UX (얼마였지? 방식)**
```
앱 아이콘 클릭
    → 즉시 카메라(스캐너)가 열림
```
총 1단계, 1~2초 소요

**왜 이게 중요한가**

마트 선반 앞에서 쓰는 앱 — 상황이 불편하고 손이 바쁨. 한 손에 물건, 한 손에 폰. 버튼 여러 번 누르는 게 귀찮으면 앱을 안 씀. **진입 마찰이 0에 가까울수록 실제 사용률이 폭발적으로 증가**한다.

참조: WeChat QR 스캔, KakaoTalk QR 스캔 모두 1~2탭으로 카메라 진입. 사용률이 높은 이유.

---

**작동 방식**

```
1. 사용자가 마트에서 샴푸 바코드 스캔
2. 앱이 자동으로 저장: [2025-12-15] [쿠팡 최저가 19,800원]
3. 89일 후 같은 샴푸를 마트에서 다시 스캔
4. 앱 표시:
   ─────────────────────────────────
   🕐 89일 전 귀하가 스캔했을 때
      온라인 최저가: 19,800원

   📌 현재 온라인 최저가: 24,500원
      → 4,700원 상승 (+23.7%)

   🏪 지금 마트 가격 (수동 입력 또는 제보): 26,900원
      → 온라인보다 2,400원 더 비쌈 (마트 세일 아님)
   ─────────────────────────────────
```

**기대 효과**

- **락인 효과**: 내 스캔 이력이 앱에 쌓일수록 이탈이 어려워짐 (내 데이터 = 내 자산)
- **감정적 연결**: 앱이 나의 쇼핑 일기가 됨
- **Price Note 대비 압도적 편의성**: 자동 기록 vs 수동 입력

### 8-4. USP 3: 2초 스캔 — 기술적으로 달성 가능한 공격적 목표

**왜 경쟁사들이 느린가**
- 순차 API 호출: A 결과 → B 호출 → B 결과 → C 호출 → ... → 30초
- 캐싱 없음: 매번 전체 데이터를 새로 가져옴

**우리의 기술 접근**

```
바코드 인식 (ML Kit): 100ms
                ↓ (동시 실행)
쿠팡 API ────────────────────→ 결과
네이버 API ──────────────────→ 결과  ← 가장 느린 것 기준 대기 (병렬)
G마켓 API ───────────────────→ 결과
                ↓
Redis 캐시 히트 시: 500ms 이하
콜드 스캔 시: 1.5~2초
```

**목표**: 캐시 기준 1초, 최대 2초 이내

## 10. 가격 추적 심리학 & 유저 리텐션 전략

### 10-1. 가격 추적이 강력한 습관을 만드는 심리학

**손실 회피 (Loss Aversion)**
> "내가 35,000원에 봤는데 지금 42,000원이면, 예전에 살 걸..."

→ 이 감정이 "다음에는 꼭 앱으로 확인해야지" 동기를 만들고 재사용으로 이어짐

**가격 앵커링 효과**
> 내가 처음 스캔했을 때 가격 = 심리적 기준점 ("원래 35,000원짜리")

→ 통계: 가격 기준점 제시 시 구매 의향 **32% 증가**
→ "원래 35,000원인데 지금 31,000원" = 실제 할인이 아니어도 좋은 가격으로 인식

**몰입 증폭 (Data Lock-in)**
> 내 스캔 이력이 앱에 쌓일수록 이탈이 더 어려워짐

→ "내 1년치 쇼핑 이력이 여기 있는데 다른 앱으로 못 가겠네"
→ 이 효과는 시간이 지날수록 강해짐 (경쟁사가 따라잡기 어려운 해자)

**도파민 루프**
```
마트에서 스캔 (트리거)
    → 즉시 결과 확인 (루틴)
    → "온라인이 더 싸네!" 발견 (보상)
    → 다음번에도 스캔하게 됨 (습관)
```

### 10-2. 리텐션 전략 — 쇼핑앱 4주 내 이탈률 86% 극복

**전략 1: 즉각적 가치 제공 (첫 스캔에서 WOW 경험)**
- 가입 없이 첫 스캔에서 즉시 최저가 비교 결과 제공
- 첫 스캔에서 온라인이 더 싸면 → "방금 OOO원 절약 기회를 발견했어요!" 표시

**전략 2: 가입 없이 바로 사용**
- Keepa 성공 사례 벤치마크: 계정 없이 즉시 사용 → 가입 마찰 없음
- 가격 이력은 로컬 저장 우선 (가입 없이도 내 기록 유지)
- 싱크/백업 원할 때만 가입 유도

**전략 3: 알림 정확도 최우선 (PriceSpy 실패 반면교사)**
> PriceSpy 실패 사례: 가격이 내려가지 않았는데도 알림 발송 → 신뢰도 붕괴

- 알림 발송 전 **반드시 실제 API로 가격 재확인** 후 발송
- 허위 알림 = 0건 목표

**전략 4: 무지출 챌린지 연동 (한국 트렌드 활용)**
- "이번 주 불필요한 지출 0원 챌린지" → 앱으로 가격비교 후 구매 결정
- 챌린지 완료 시 SNS 공유 → 바이럴

**전략 5: 개인화 절약 리포트 (월간)**
- 매월 1일: "지난 달 귀하의 절약 리포트"
- 총 스캔 횟수, 온라인 vs 마트 가격 차이 통계
- 공유 가능한 카드 형태 → 카카오톡/인스타그램 공유 → 바이럴

## 최종 결론 — 왜 지금 "얼마였지?"인가

1. **네이버가 만든 시장 공백**: 한국 최대 가격비교 플랫폼이 2024년 기능을 제거 → 수천만 사용자가 갈 곳을 잃음

2. **경쟁사들의 해결 안 된 문제**: 15년이 지나도 30초 걸리는 속도, 20% 오류율 — 기술적으로 해결 가능

3. **Price Note가 증명한 니즈**: 가격을 기록하고 싶은 수요는 실재한다. 우리는 그 허들을 "바코드 스캔 자동화"로 제거한다

4. **전 세계에 없는 기능**: "내가 스캔했을 때 가격" 자동 기록 — 완전한 블루오션

5. **명확한 대중 메시지**: "마트 가짜 세일에 속지 마세요" — 설명이 필요 없는 직관적 가치

6. **타이밍**: 인플레이션으로 가격 민감도가 역사상 가장 높은 시기 — 바로 지금
